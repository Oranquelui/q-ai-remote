# Q AI Remote 紹介文テンプレート (日本語)

## 1. 一行紹介
Telegramだけで、ローカルPCの安全なファイル作業を `Plan -> 承認 -> 実行` で回せるAI運用ツール。

## 2. GitHub README 冒頭向け (短文)
Q AI Remote は、TelegramをUIとして使う安全ガバナンス型のAI作業ツールです。  
任意コマンド実行を禁止し、`list_dir/read_file/create_file/patch_file` のみを許可。  
変更は必ず承認フローを通し、実行後は `diff + JSONL + HTML` を監査保存します。

## 3. 配布ページ向け (中文)
Q AI Remote は「便利さ」と「事故防止」を両立するために作られました。  
ユーザーはTelegramから自然文で依頼できますが、実行は必ず `Plan+Risk` を確認してから。  
Allowed Paths外アクセス、任意コマンド実行、危険なネットワーク操作はブロックされます。  
実行後は差分と監査ログがローカルに残るため、後から追跡・検証が可能です。  
個人開発から小規模チーム運用まで、まずは安全なMVPとして導入できます。

## 4. X投稿向け (3パターン)
### 4-1. ローンチ告知
Q AI Remote を公開しました。  
Telegramで依頼 -> Plan+Risk確認 -> 承認後だけ実行。  
任意コマンド禁止、実行ログは diff/JSONL/HTML で保存。  
「使いやすさ」と「安心」を両立するAI運用ツールです。

### 4-2. セキュリティ訴求
便利なAI自動化は、実行ガードがないと危険です。  
Q AI Remote は `Allowed Paths` 強制 + 承認必須 + 監査ログ標準搭載。  
Telegram操作でもRCEっぽく見えない運用を目指しました。

### 4-3. 開発者向け
ローカル開発をTelegramで遠隔操作したい人向け。  
Q AI Remote は `/task -> Plan+Risk -> ✅実行` の運用に固定。  
MVPでも運用事故を防ぐ設計を優先しています。

## 5. 禁止事項の説明テンプレ
- 任意コマンド実行はできません。  
- Allowed Paths 外には書き込めません。  
- 承認前に実行はされません。  
- 実行後は監査ログを必ず記録します。

## 6. OpenClawを知っている人向けの説明
OpenClawの「遠隔で使える体験」の良さを参考にしつつ、  
Q AI Remote は配布初期段階での事故防止を優先し、承認・制限・証跡を強めています。  
まずは安全運用のMVPとして導入し、段階的に自由度を上げる前提です。
